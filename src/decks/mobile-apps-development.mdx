### âŒ¨ï¸ Widgets: Stateful vs Stateless

---

### Stateful Widget

- [Interactive](https://flutter.dev/docs/development/ui/interactive)
- Has in internal State than can be altered
  - **setState** to change it
- Has Lifecycle
  - **initState()**: called when added to the app tree
  - **dispose()**: called when removed from the Widgets tree
- Implementation
  - State Class
  - Layout Class that extends the State Class

---

### Stateless Widget

- Immutable

  - No dynamic internal State
  - Lot of built-in mechanism to trigger UI reloads
  - Properties defined inside will never change

- Implementation

  - Implement a `build` Method, give it a `context` to say where it lives in the Widgets Tree

- Properties are defined inside the constructor
  - `key` is used to let flutter know the order and stuff
