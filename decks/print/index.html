🤷‍♀️ What is Expo

  • Framework and a platform for universal React applications
  • Brings a set of tools built around React Native and native platforms

⌨️ Widgets: Stateful vs Stateless

Stateful Widget

  • Interactive
  • Has in internal State than can be altered
    • setState to change it
  • Has Lifecycle
    • initState(): called when added to the app tree
    • dispose(): called when removed from the Widgets tree
  • Implementation
    • State Class
    • Layout Class that extends the State Class

Stateless Widget

  • Immutable

    • No dynamic internal State
    • Lot of built-in mechanism to trigger UI reloads
    • Properties defined inside will never change

  • Implementation

    • Implement a build Method, give it a context to say where it lives in the Widgets Tree

  • Properties are defined inside the constructor

    • key is used to let flutter know the order and stuff

A UI toolkit, to:

  • Build mobile and desktop apps
  • Using Web Technologies for layout and logic (HTML,CSS, JS)

Max Lynch, Ben Sperry, and Adam Bradley of Drifty Co.

🥅 Main Goal

Be truly:

  • Cross-platform with one code base
    • iOS
    • Android
    • Desktop
    • PWA
    • Progressive Web App
  • Simple
    • By using web standards

🧰 Setup

  • Install NodeJS
  • Install VSCode or any other editor

🛠 Choose a Framework

  • React flavoured Ionic
npm install -g ionic@latest
ionic start MyApp blank --type=react
  • Angular flavoured Ionic
npm install -g ionic native-run cordova-res
ionic start MyApp tabs --type=angular --capacitor

🛠 Want to run it on Native Platform

Run it on native platform: Full steps

ionic capacitor add android # Generate the native project
# Set package id inside `capacitor.config.json` => appId
ionic capacitor open ios # Make sure it matches inside XCode
ionic capacitor run ios -l --address=0.0.0.0 # Run it

  • node_modules/: you package dependencies

  • ionic.config.json: app configuration
  • package.json: dependencies listing, other configs
